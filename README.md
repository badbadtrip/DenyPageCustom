## Описание

При запуске сервера плагин читает параметры из `init.conf` и генерирует готовый JS-файл с полноэкранной страницей входа. Файл перезаписывается автоматически при каждом обновлении конфига — редактировать его вручную не нужно.

---

## Возможности

- **Полная кастомизация текстов** — заголовок, подзаголовок, шаги инструкции, подписи QR-блока
- **QR-код Telegram** — автоматически генерируется для любого Telegram-бота/канала; отображается только при наличии `tg_target` и `show_qr: true`
- **Горячая перезагрузка** — `"dynamic": true` позволяет применять изменения без перезапуска сервера
- **Хэш-диффинг** — файл записывается только при реальных изменениях конфига
- **Защита от XSS** — все строки из конфига экранируются через `JsonSerializer.Serialize`
- **Адаптивный дизайн** — корректно отображается на TV, планшетах и мобильных устройствах

## Архитектура

```
DenyPageCustom/
├── manifest.json          # Метаданные плагина для Lampac
├── ModInit.cs             # Точка входа: IModuleLoaded, таймер, подписка на события
├── DenyPageGenerator.cs   # Генератор JS-файла (чистый static builder)
└── Models/
    └── DenyPageConf.cs    # Модель конфигурации с дефолтами
```

### Компоненты

**`ModInit.cs`** — точка входа, реализует `IModuleLoaded`:
- При загрузке создаёт директорию вывода и вызывает `SyncAndGenerate()` один раз
- Подписывается на событие `EventListener.UpdateInitFile` (обновление `init.conf`)
- Запускает таймер с интервалом 3 секунды как резервный механизм синхронизации

**`DenyPageGenerator.cs`** — статический построитель:
- Метод `Build(DenyPageConf)` возвращает полное содержимое `deny.js` в виде строки
- Все JS-строки санируются через вспомогательный метод `Js()` (`JsonSerializer.Serialize`)
- Правая колонка с QR-кодом генерируется только если `tg_target` задан и `show_qr = true`

**`Models/DenyPageConf.cs`** — модель конфигурации:
- Заполняется через `ModuleInvoke.Init("DenyPage", new DenyPageConf())` из `init.conf`
- Все поля имеют значения по умолчанию; пустые строки получают русскоязычные дефолты внутри `Build()`

## Установка

Lampac NextGen использует **Roslyn** — `.cs` файлы компилируются сервером автоматически при старте. Предварительная сборка в DLL не требуется.

1. Скопируйте папку `DenyPageCustom` (со всеми `.cs` файлами и `manifest.json`) в директорию `mods/` рядом с исполняемым файлом Lampac:
   ```
   mods/
   └── DenyPageCustom/
       ├── manifest.json
       ├── ModInit.cs
       ├── DenyPageGenerator.cs
       └── Models/
           └── DenyPageConf.cs
   ```
2. Перезапустите сервер — Roslyn скомпилирует модуль автоматически. Скомпилированная сборка кэшируется; при последующих запусках без изменений исходников перекомпиляция не происходит.
3. При `"dynamic": true` (уже установлено в `manifest.json`) сервер следит за изменениями `.cs` файлов и перекомпилирует модуль **без перезапуска**.

Файл `plugins/override/deny.js` будет создан автоматически после загрузки модуля.

## Конфигурация

Все параметры задаются в секции `[DenyPage]` файла `init.conf`:

> Изменения применяются автоматически — перезапуск сервера не требуется.
```json
"DenyPage": {                            
    "tg_target":"@YourBot",
    "show_qr": true,
    "page_badge":"NextGen",                                 
    "page_title": "Вход в систему",
    "page_subtitle": "Для доступа к серверу введите пароль, выданный администратором.",
    "step1_text": "Введите пароль в поле ниже и нажмите «Войти».",
    "step2_text": "Если пароля нет — напишите боту <b>@YourBot</b>.",
    "qr_caption": "Нет пароля?",
    "qr_subcaption": "Отсканируй QR или нажми кнопку, чтобы написать боту.",
    "tg_button_text": "Написать боту"
  }
```

## Поля конфигурации

| Поле                   | Тип      | Умолчание                                        | Статус                                                                            |
| ---------------------- | -------- | ------------------------------------------------ | --------------------------------------------------------------------------------- |
| `tg_target`            | `string` | `""`                                             | Управляет QR-блоком. Принимает `@username`, `https://t.me/...` или `tg://` ссылку |
| `show_qr`              | `bool`   | `true`                                           | QR-блок отображается только если `tg_target` задан **и** `show_qr = true`         |
| `page_title`           | `string` | `"Вход в систему"`                               | Заголовок страницы — отрисовывается                                               |
| `page_subtitle`        | `string` | `"Для доступа к серверу введите пароль..."`      | Подзаголовок — отрисовывается                                                     |
| `step1_text`           | `string` | `"Введите пароль в поле ниже..."`                | Текст шага 01 — отрисовывается                                                    |
| `step2_text`           | `string` | Авто (с упоминанием бота если `tg_target` задан) | Текст шага 02 — отрисовывается. Поддерживает `<b>`                                |
| `qr_caption`           | `string` | `"Нет пароля?"`                                  | Заголовок QR-блока — отрисовывается                                               |
| `qr_subcaption`        | `string` | `"Отсканируй QR или нажми кнопку..."`            | Подпись QR-блока — отрисовывается                                                 |
| `page_badge`           | `string` | `"Lampac NextGen"`                               | Значение вычисляется, HTML-элемент удалён                                         |
| `hint_text`            | `string` | `""`                                             | Значение вычисляется, HTML-элемент удалён                                         |
| `footer_text`          | `string` | `""`                                             | Значение вычисляется, HTML-элемент удалён                                         |
| `tg_button_text`       | `string` | `"Открыть Telegram"`                             | Текст кнопки Telegram — используется из конфига                                   |
| `qr_size`              | `int`    | `200`                                            | В модели; не используется — размер QR фиксирован (400px / 180px на TV)            |
| `show_password_button` | `bool`   | `true`                                           | В модели; не используется в генераторе                                            |
| `show_cub_button`      | `bool`   | `false`                                          | В модели; не используется в генераторе                                            |
| `show_please_wait`     | `bool`   | `false`                                          | В модели; не используется в генераторе                                            |
| `instruction_url`      | `string` | `""`                                             | В модели; не используется в генераторе                                            |
| `instruction_text`     | `string` | `""`                                             | В модели; не используется в генераторе                                            |

### Форматы `tg_target`

Все варианты нормализуются к корректной ссылке автоматически:

```
@mybotname          →  https://t.me/mybotname
mybotname           →  https://t.me/mybotname
https://t.me/bot    →  https://t.me/bot       (без изменений)
tg://resolve?...    →  tg://resolve?...        (без изменений)
```

## Логика авторизации

Сгенерированный `deny.js` внедряет две функции:

### `checkAutch()`
Вызывается немедленно при загрузке страницы. Отправляет GET-запрос на `{localhost}/testaccsdb` с текущими данными устройства. Если ответ содержит `res.accsdb = true`:
- Скрывает UI приложения (`#app`)
- Устанавливает `window.sync_disable = true`
- Через 500 мс вызывает `addDevice()`

### `addDevice(message)`
Рендерит полноэкранную форму входа в `document.body`. Обрабатывает логин:

```
POST {localhost}/testaccsdb?account_email=<пароль>&uid=<uid>
```

| Результат                                      | Поведение                                                                             |
| ---------------------------------------------- | ------------------------------------------------------------------------------------- |
| `result.success = true`, `result.uid` задан    | Создан новый аккаунт — показывает сгенерированный пароль, редирект на `/` через 3 сек |
| `result.success = true`, `result.uid` не задан | Сохраняет `lampac_unic_id` в storage, редирект на `/`                                 |
| `result.success = false`                       | Показывает «Неправильный пароль», снимает блокировку кнопки                           |
| Ошибка сети                                    | Показывает «Ошибка соединения», снимает блокировку кнопки                             |

> QR-изображение загружается в runtime с `api.qrserver.com` — не бандлится в плагин.

## Адаптивная вёрстка

| Разрешение / Условие              | Поведение                                                                                                      |
| --------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| TV (≥ 1400px)                     | Макс. ширина 1100px, высота 85vh, увеличенные отступы, QR-код 180×180px для лучшей видимости на большом экране |
| Desktop (> 900px)                 | Двухколоночный макет: форма слева (max-width 960px), QR-блок справа (ширина 280px)                             |
| Планшет (700–900px)               | Плавное сжатие: max-width 720px, уменьшенные padding                                                           |
| Мобильный (≤ 700px)               | Одна колонка: QR-блок над формой, строчная раскладка внутри блока                                              |
| Мобильный ландшафт (h ≤ 500px)    | Горизонтальная раскладка с прокруткой контента                                                                 |
| Маленький мобильный (≤ 420px)     | Минимальные отступы (24px/20px) для экономии места                                                             |

## Разработка

### Добавление нового поля конфигурации

1. Добавьте свойство в `Models/DenyPageConf.cs` с дефолтным значением
2. Прочитайте значение в `DenyPageGenerator.Build()` и включите его в генерируемый HTML/JS
3. Обновите таблицу полей в этом README

### Структура сгенерированного файла

```
deny.js
├── CSS (инлайн, через <style>)
├── function addDevice(message)   — форма входа + обработчики
└── function checkAutch()         — проверка авторизации + вызов addDevice
    └── checkAutch();             — немедленный вызов
```

### Безопасность

- Все значения из конфига передаются в JS через `JsonSerializer.Serialize` — экранирование кавычек и спецсимволов гарантировано
- XSS через `step2_text` возможен намеренно (поле поддерживает `<b>` теги) — контролируется администратором через `init.conf`

## manifest.json

```json
{
  "name": "DenyPageCustom",
  "version": "1.1",
  "enable": true,
  "dynamic": true
}
```

`"dynamic": true` — плагин загружается без перезапуска сервера Lampac.
